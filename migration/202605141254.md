## **2. Architectural Model**

Xanadu Prime is built on a **three‑layer architecture**:

1. **Substrate Layer** — the ontological foundation
2. **Governance Layer** — the rules that prevent drift
3. **Corpus Layer** — the documents that form the Prime epoch

These layers are **strictly separated** and **mutually reinforcing**.

## **3. Substrate Layer**

The substrate defines the **non‑negotiable invariants** of the system:

- **Permanent Addressing** — every surface has a stable, immutable address
- **Deep Versioning** — no overwrites; every change creates a new version
- **Bidirectional Linking** — every link is mutual and traceable
- **Transclusion Integrity** — quoted content is never duplicated
- **Immutable History** — no retroactive edits

These invariants ensure that Prime is **stable without being static**.

## **4. Governance Layer**

The governance layer enforces **coherence, stability, and non‑drift** across the system.  
It consists of:

- **DSR‑1 (Decision Stability Rule)** — all decisions must be declared, scope‑bound, and stabilized
- **Razor Principles** — contradictions must be reconciled, split, or elevated
- **Naming Governance Clause** — names must be unique, stable, and lineage‑consistent
- **Lifecycle Rules** — surfaces move from draft → stabilized → published

This layer ensures that Prime evolves **intentionally**, not implicitly.

## **6. Addressing Model**

Prime uses a **permanent addressing model**:

- every surface receives a **stable identifier**
- identifiers never change
- links always resolve
- no surface is ever overwritten
- new versions receive new addresses

This eliminates link rot and ensures **perpetual referential integrity**.

## **7. Versioning Model**

Prime implements **deep versioning**:

- every edit produces a new version
- previous versions remain accessible
- no version is ever deleted
- lineage is explicit and traceable
- transclusions always reference specific versions

This creates a **complete historical record** of the corpus.

**Provenance Timeline**

## **8. Transclusion Model**

Prime supports **true transclusion**, meaning:

- content is referenced, not copied
- the original source remains authoritative
- transclusions update automatically when the source changes
- every transclusion is bidirectionally linked

This ensures **integrity, consistency, and non‑duplication** across the corpus.

## **9. Lineage Model**

Prime enforces **explicit lineage**:

- every surface has a parent
- every version has an ancestor
- every successor must derive from Prime
- no implicit lineage is allowed
- no retroactive changes are permitted

Prime is the **root** of the operational lineage.
