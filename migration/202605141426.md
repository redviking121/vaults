```markdown
# Library Note Factory — Plugin Contract  
Version: 1.0.0  
Status: Authoritative

---

## Purpose

Define the authoritative contract for the Library Note Factory plugin:

- Scope: What the plugin is allowed to do.
- Interface: What Layer‑42 and Layer‑43 can rely on.
- Membranes: How it interacts with Templater and the Library.
- Stability: What must not change without a versioned contract update.

This document governs implementation, refactors, and future extensions.

---

## Scope of authority

The plugin has narrow, explicit authority:

- File creation:  
  Create new Library notes for known classes using stable filename patterns.

- Command surface:  
  Expose Layer‑43 commands with stable IDs and predictable behavior.

- Metadata handoff:  
  Pass class metadata to Templater exactly as defined in the class appendix.

- Event emission:  
  Emit structured events for every command invocation.

---

## Explicit non-authority

The plugin does **not**:

- Generate content  
- Enforce taxonomy rules  
- Manage archives  
- Perform batching logic  
- Modify existing notes  
- Resolve naming conflicts beyond the defined collision rules

---

## Membrane: Plugin ↔ Layer‑42

- Layer‑42 defines intent classes.  
- Plugin resolves intent → class → filename → creation.  
- Plugin must not reinterpret or mutate Layer‑42 definitions.

---

## Membrane: Plugin ↔ Templater

- Plugin creates the file.  
- Plugin calls the correct Templater command.  
- Templater injects content and metadata.  
- Plugin must not modify injected content.

---

## Event model

Every command emits:

- `L43.CMD.<COMMAND_ID>.INVOKED`
- `L43.CMD.<COMMAND_ID>.RESULT.<OK|ERROR>`

Events include:

- `user`
- `workspace`
- `plugin_id`
- `timestamp`
- `correlation_id`

---

## Filename and UID guarantees

- Filenames follow the class appendix patterns.  
- UIDs are generated using the stable UID generator.  
- Collisions follow the documented collision behavior.

---

## Error semantics

- All errors are non-fatal.  
- Plugin must fail closed, never generating partial notes.  
- User-facing notifications are terse and singular.

---

## Lifecycle hooks

- `onload`: register commands  
- `onunload`: unregister commands  
- No side effects outside Library

---

## Stability guarantees

- Command IDs remain stable across versions.  
- Filename patterns remain stable.  
- Metadata schema remains stable.  
- Any change requires a versioned contract update.

Implementation may evolve; the contract is the law.
```

```markdown
- **Filename:** `Note - {{uid}}.md`
- **Required metadata:**  
  - `uid`  
  - `class: note`  
  - `created`  
- **Optional metadata:**  
  - `tags`

  `{ class: "note", uid, created }`
- **Collision behavior:**  
  Regenerate UID and retry.
- **Layer‑42 mapping:**  
  `intent.note.create`

---

## Class: Task

- **Filename:** `Task - {{uid}}.md`
- **Required metadata:**  
  - `uid`  
  - `class: task`  
  - `created`  
  - `status: open`
- **Templater payload:**  
  `{ class: "task", uid, created, status: "open" }`
- **Collision behavior:**  
  Same as Note.
- **Layer‑42 mapping:**  
  `intent.task.capture`

---

## Class: Signal

- **Filename:** `Signal - {{timestamp}} - {{uid}}.md`
- **Required metadata:**  
  - `uid`  
  - `class: signal`  
  - `timestamp`
- **Templater payload:**  
  `{ class: "signal", uid, timestamp }`

  Append `-1`, `-2`, etc.
```

```markdown
  - Command palette
  - Keybindings (hot path only)
  - Context menus

---

### Core
- `l43.core.help` — Show plugin help  
- `l43.core.reload` — Reload plugin  
- `l43.core.log` — Open event log

### Capture
- `l43.capture.note` — Create Note  
- `l43.capture.task` — Create Task  
- `l43.capture.signal` — Create Signal

### Navigation
- `l43.nav.library` — Jump to Library root  
- `l43.nav.operators` — Jump to Operators folder  
- `l43.nav.recent` — Jump to recent Library notes

- `l43.review.daily` — Open daily review  
- `l43.review.weekly` — Open weekly review

- `user`
- `workspace`
```
