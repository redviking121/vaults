# 🟥 1. What Execution semantics _is_

Execution semantics is the **behavioral law** that determines:

- how the entity **starts**
- how it **executes**
- how it **yields or blocks**
- how it **handles state transitions**
- how it **interacts with the substrate**
- how it **terminates or persists**

It is the **deterministic backbone** of runtime behavior.

This sits directly beneath **Lifecycle semantics** and above **Readiness semantics**.

# 🟥 2. The four canonical surfaces of Execution semantics

Each entity expresses execution through four discriminative surfaces:

- **Invocation_rules** — when and how the entity may be called
- **Transition_rules** — how it moves between internal states
- **Concurrency_rules** — how it behaves under parallelism or contention
- **Termination_rules** — how it ends, yields, or hands off control

These four surfaces form the **execution grammar**.

---

# 🟥 3. Invocation rules

This surface defines:

- allowed entry points
- required preconditions
- forbidden states
- argument constraints
- substrate‑level gating

It is the membrane that ensures the entity is only invoked under **valid, stable conditions**.

This is where Execution semantics touches **Readiness semantics**.

---

# 🟥 4. Transition rules

This surface defines:

- legal state transitions
- forbidden transitions
- transition guards
- stabilization requirements
- rollback or retry behavior

This is the **state machine spine** of the entity.

It is the runtime expression of the Xi/Razor **Decision Stability Rules (DSR)**.

# 🟥 5. Concurrency rules

This surface defines:

- whether the entity is **exclusive**, **shared**, or **reentrant**
- how it handles **contention**
- how it behaves under **parallel execution**
- whether it can be **interrupted**, **paused**, or **resumed**

This is the membrane that prevents **semantic race conditions**.

It is the execution‑level counterpart to **Structural_signature**.

---

# 🟥 6. Termination rules

This surface defines:

- how the entity completes
- what constitutes a “successful” termination
- what happens on failure
- what state is emitted
- whether cleanup is required
- whether the entity persists or dissolves

This is the **exit grammar** of the entity.

It is the runtime expression of **Error semantics**.
