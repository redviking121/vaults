# 🟥 3. The Tool trait implementation

This is where the struct becomes a DeepSeek Tool:

```
#[async_trait]
impl Tool for InventoryScanTool {
    fn name(&self) -> &'static str {
        "inventory_scan"
    }

    fn description(&self) -> &'static str {
        "Runs a host-first software and system inventory scan and returns structured JSON."
    }

    fn schema(&self) -> Value {
        json!({
            "type": "object",
            "properties": {
                "filter": {
                    "type": "string",
                    "description": "Optional filter for narrowing scan results."
                }
            },
            "required": []
        })
    }

    async fn run(&self, args: Value) -> ToolResult {
        let filter = args.get("filter").and_then(|v| v.as_str()).map(|s| s.to_string());
        let result = self.scanner.scan(filter).await?;
        Ok(json!(result))
    }
}
```

This is the **Tool_trait_binding**.

Key surfaces:

- **name()** → `"inventory_scan"` is the stable call‑sign
- **description()** → what the model uses to decide when to call it
- **schema()** → argument contract
- **run()** → host execution boundary

This is the membrane where Inventory becomes callable.
