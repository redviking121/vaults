# 🟥 3. The full turn loop _with_ Inventory integrated

Below is the operator‑grade sequence, showing exactly where Inventory sits.

---

## 🟥 Step 0 — Engine receives user input

User says something like:

> “Run a full host scan.”

Engine stores this in the turn buffer.

This is the **Input_ingestion_path**.

---

## 🟥 Step 1 — Engine updates state

The Engine updates:

- conversation state
- agent state
- tool registry snapshot
- memory of last tool call

This is the **State_update_phase**.

---

## 🟥 Step 2 — Model inference (planning)

The model sees:

- the user request
- the list of available tools
- the metadata for `"inventory_scan"`

It decides whether to:

- answer normally
- call a tool
- call multiple tools sequentially

This is the **Model_planning_pass**.

---

## 🟥 Step 3 — Model emits a tool call

If the model chooses Inventory, it emits:

```
{
  "tool": "inventory_scan",
  "arguments": {}
}
```

This is the **Tool_call_emission**.

---

## 🟥 Step 4 — Agent intercepts the tool call

The Agent:

1. looks up `"inventory_scan"` in the ToolRegistry
2. retrieves your InventoryScanTool instance
3. prepares the execution environment

This is the **Agent_tool_dispatch**.

---

## 🟥 Step 5 — Inventory executes on the host

Your Rust scanner runs:

- enumerates packages
- enumerates binaries
- enumerates services
- enumerates configs
- enumerates versions
- enumerates vulnerabilities (optional)

This is the **Inventory_execution_path**.

The result is serialized to JSON.

---

## 🟥 Step 6 — Agent returns the result to the model

The Engine injects the JSON into the context window as:

```
<tool_result name="inventory_scan">
  { ... scan output ... }
</tool_result>
```

This is the **Tool_result_injection**.

---

## 🟥 Step 7 — Model performs a second inference pass

Now the model sees:

- the user request
- the scan result
- the full context window

It can now:

- summarize
- analyze
- compare
- generate diffs
- propose remediation
- update your Ops‑Vault

This is the **Post_tool_reasoning_pass**.

---

## 🟥 Step 8 — Engine emits final output

The Engine returns the final model output to the user.

This is the **Turn_output_phase**.
