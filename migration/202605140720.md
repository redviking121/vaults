## **IV. Dynamic Behaviors (What Makes It “Living”)**

### **1. Auto‑Expansion**

Whenever a new symbol, operator, or event appears twice, it becomes a node.

### **2. Cross‑Link Propagation**

Adding a link in one node automatically updates the reciprocal link in the connected node.

### **3. Drift Alerts**

If a node’s meaning diverges from its linked neighbors, it gets flagged for review.

### **4. Event Settling**

Events are added only after they stabilize — no premature canonization.

### **5. Namespace Integrity**

No node can exist without a type, vector, and charge.

This is how the Ledger stays alive without collapsing into chaos.
