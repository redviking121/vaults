# **Axioms → Linter Rule Mapping (v1.0)**

**Namespace:** `LN-*`  
**Status:** Authoritative  
**Purpose:** Translate AX‑series axioms into concrete, enforceable linter rules.

Each AX rule maps to one or more LN rules.  
Each LN rule includes:

- **Check:** What the linter must verify
- **Failure Condition:** What constitutes a violation
- **Scope:** Files, folders, frontmatter, links, IDs, or global invariants
- **Severity:** `error`, `warning`, or `info`

---

# **AX‑01 → LN Rules**

### **AX‑01 — Single Source of Truth**

**LN‑01.1 — Duplicate Canonical Objects**

- **Check:** No two notes declare the same canonical identity (same concept ID, person ID, project ID).
- **Failure:** Two or more files contain identical `id:` fields.
- **Severity:** error

**LN‑01.2 — Duplicate Facts**

- **Check:** No persistent fact appears in more than one canonical home.
- **Failure:** Same fact sentence appears in multiple notes without explicit reference.
- **Severity:** warning
