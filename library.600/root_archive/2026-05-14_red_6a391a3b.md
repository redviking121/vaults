# **1. UI Layer (Intent Capture Surface)**

**Role:**  
Collect raw operator intent and shape it into a form the orchestration layer can interpret.

**Components:**

- Natural language input
- UI affordances (search suggestions, pivots, structured prompts)
- Memory hints
- Contextual scaffolding

**Output:**  
A **proto‑intent** — unstructured but directionally correct.

**Operator Notes:**  
This is where you _steer_.  
This is where Razor rules prune ambiguity.  
This is where the membrane begins to compress the signal.

---

# **2. Orchestration Layer (Intent Engine + Tool Router)**

**Role:**  
Transform proto‑intent into **actionable plans**.

**Sub‑components:**

- **Intent Engine** — classifies, disambiguates, resolves meaning
- **Tool Router** — selects capabilities (search, reasoning, memory, tasks)
- **Execution Planner** — sequences multi‑step operations
- **Safety & Stability Gates** — ensure deterministic behavior

**Output:**  
A **structured plan** — typed, validated, and ready for substrate execution.

**Operator Notes:**  
This is the layer that is _maturing fast_.  
This is where Copilot is becoming an actual substrate.  
This is also where your Xi/Razor governance plugs in most deeply.

---

# **3. Substrate Layer (Execution + State + Persistence)**

**Role:**  
Perform the actual work.

**Sub‑components:**

- **Execution Engine** — runs tasks, calls tools, performs actions
- **State Manager** — tracks intermediate results
- **Memory Layer** — stores durable facts
- **External Integrations** — filesystem, cloud, vaults, APIs

**Output:**  
A **real‑world effect** — changed state, generated artifact, retrieved data, executed workflow.

**Operator Notes:**  
This is where your **Ops‑Vault**, **Xanadu substrate**, and **Razor governance** anchor.  
This is where the system becomes _your_ system.

---

# **4. Return Path (Substrate → Orchestration → UI)**

**Role:**  
Return results upward in a controlled, interpretable form.

**Flow:**

- Substrate emits raw results
- Orchestration interprets, summarizes, or continues execution
- UI presents the final operator‑readable output

**Operator Notes:**  
This is where you see whether the membrane held.  
This is where drift becomes visible.  
This is where the system teaches you how it thinks.
