# 🟥 1. What Parameter existence _is_

Parameter existence is the membrane that validates the **presence** of all required inputs before an entity may begin.

It enforces:

- required parameters must exist
- required fields must be present
- required values must not be null
- required structures must not be empty

If a required parameter is absent, the entity cannot even enter **Readiness_semantics**.

This is the substrate’s protection against **undefined invocation**.

# 🟥 2. The four discriminative surfaces of Parameter existence

Each parameter existence check falls into one of four discriminators:

- **Presence_checks** — the parameter must exist
- **Non_null_checks** — the parameter must not be null
- **Non_empty_checks** — the parameter must not be empty
- **Structural_presence_checks** — required subfields must exist

These four surfaces form the **input spine** of the entity.

---

# 🟥 3. Presence checks

Presence checks enforce the most basic rule:

> _“The parameter must exist in the invocation.”_

If the parameter is missing, the entity is **illegal to instantiate**.

Presence checks prevent **phantom invocation** — calling an entity without the required inputs.

---

# 🟥 4. Non‑null checks

Non‑null checks enforce:

> _“The parameter exists, but it must not be null.”_

This prevents **null‑poisoning**, where a parameter exists syntactically but is semantically unusable.

Null is treated as **non‑existence**.

---

# 🟥 5. Non‑empty checks

Non‑empty checks enforce:

> _“The parameter exists and is non‑null, but it must also contain content.”_

This applies to:

- strings
- lists
- maps
- buffers
- collections

An empty structure is treated as **semantic absence**.

---

# 🟥 6. Structural presence checks

Structural presence checks enforce:

> _“The parameter exists, is non‑null, non‑empty, and its required subfields must also exist.”_

This is the membrane that prevents **partial structures** from corrupting the substrate.

It ensures that composite parameters are **fully formed** before invocation.

- safe invocation

- predictable composition

It is the **first membrane** that protects the entity from undefined behavior.

Without parameter existence, the substrate becomes **unstable**, **ambiguous**, and **corruptible**.

With it, the substrate becomes **deterministic**, **safe**, and **composable**.
