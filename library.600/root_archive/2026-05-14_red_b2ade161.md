# 🟥 1. What Lifecycle semantics _is_

Lifecycle semantics defines:

- the **phases** an entity can occupy
- the **conditions** for entering or exiting each phase
- the **stability requirements** before transitions
- the **propagation rules** for state
- the **dissolution or persistence** logic

It is the membrane that ensures the entity’s existence is **coherent, predictable, and schedulable** across time.

This sits directly above **Execution semantics** and directly beneath **Readiness semantics**.

# 🟥 2. The four canonical lifecycle phases

Every entity expresses its lifecycle through four discriminative phases:

- **Instantiation** — the entity is created, declared, and bound to its substrate
- **Activation** — the entity becomes eligible for execution
- **Operation** — the entity performs its functional role
- **Resolution** — the entity completes, dissolves, or persists

These four phases form the **temporal grammar** of the entity.

# 🟥 3. Instantiation phase

This phase defines:

- how the entity is **declared**
- how it is **constructed**
- how its **initial state** is formed
- how it binds to its **structural family**
- how it acquires its **Field_presence_vector**

Instantiation is the **birth membrane** — the moment the entity becomes a thing.

This is where Lifecycle semantics touches **Structural_signature**.

---

# 🟥 4. Activation phase

This phase defines:

- readiness gating
- preconditions for execution
- stabilization requirements
- dependency resolution
- substrate‑level validation

Activation is the **eligibility membrane** — the entity is not running yet, but it is _allowed_ to run.

This is the lifecycle expression of **Readiness semantics**.

---

# 🟥 5. Operation phase

This phase defines:

- how the entity performs its **core function**
- how it handles **state transitions**
- how it interacts with the substrate
- how it expresses **Execution semantics**
- how it maintains **temporal stability**

Operation is the **active membrane** — the entity is doing the thing it exists to do.

This is the lifecycle expression of **Capability role**.

---

# 🟥 6. Resolution phase

This phase defines:

- how the entity completes
- how it emits final state
- how it cleans up or persists
- how it dissolves or transitions to a successor
- how errors propagate or are contained

Resolution is the **exit membrane** — the entity’s temporal arc concludes.

This is the lifecycle expression of **Error semantics**.
