# Axiom Query Language (AQL) v1.0

**Namespace:** `AQL-*`  
**Status:** Authoritative  
**Owner:** Red Horne  
**Scope:** Querying AX‑series, LN‑series, and Library objects  
**Purpose:** Provide a **non‑semantic**, **structural** query language for inspection, linting, and enforcement.

AQL is:

- **read‑only**
- **deterministic**
- **non‑narrative**
- **non‑interpretive**

---

## AQL‑01 — Core model

### AQL‑01.1 — Object model

All queries operate over **objects** with fields:

- **id:** string
- **type:** enum (`axiom`, `rule`, `note`, `index`, `link`, `meta`)
- **location:** path
- **layer:** enum (`substrate`, `operator`, `meta`)
- **status:** enum (`draft`, `active`, `deprecated`, `blocked`)
- **tags:** list
- **depends_on:** list of ids
- **links:** list of ids
- **integrity:** enum (`pass`, `fail`, `unknown`)

(AQL does not define new fields; it only addresses existing ones.)

---

## AQL‑02 — Basic syntax

### AQL‑02.1 — Query shape

General form:

```aql
SELECT <projection>
FROM <source>
WHERE <predicate>
OPTIONS <options>
;
```

All clauses except `FROM` are optional.

### AQL‑02.2 — Sources

- **AXIOMS** — all AX‑series objects
- **RULES** — all LN‑series objects
- **NOTES** — all Library notes
- **ALL** — union of all above

Example:

```aql
FROM AXIOMS;
```

---

## AQL‑03 — Projections

### AQL‑03.1 — Projection forms

- `*` — full object
- `FIELD(field-name)` — single field
- `FIELDS(field1, field2, …)` — subset

Examples:

```aql
SELECT * FROM RULES;

SELECT FIELD(id) FROM AXIOMS;

SELECT FIELDS(id, status, layer) FROM NOTES;
```

---

## AQL‑04 — Predicates

### AQL‑04.1 — Comparison operators

- `=` — equality
- `!=` — inequality
- `IN (…)` — membership
- `CONTAINS` — list membership

Examples:

```aql
WHERE status = active

WHERE layer IN (operator, meta)

WHERE tags CONTAINS structure
```

### AQL‑04.2 — Logical operators

- `AND`
- `OR`
- `NOT`

Example:

```aql
WHERE status = active AND integrity = fail
```

---

## AQL‑05 — Structural predicates

These are the **AEP/LN‑aware** predicates.

### AQL‑05.1 — Link predicates

```aql
HAS_LINK(id)
HAS_INBOUND(id)
HAS_OUTBOUND(id)
```

Examples:

```aql
WHERE HAS_LINK(AX-01)

WHERE NOT HAS_OUTBOUND(LN-12)
```

### AQL‑05.2 — Dependency predicates

```aql
DEPENDS_ON(id)
DEPENDED_BY(id)
HAS_CYCLE()
```

Examples:

```aql
WHERE DEPENDS_ON(LN-12)

WHERE HAS_CYCLE()
```

### AQL‑05.3 — Integrity predicates

```aql
INTEGRITY_PASS()
INTEGRITY_FAIL()
INTEGRITY_UNKNOWN()
```

Examples:

```aql
WHERE INTEGRITY_FAIL()
```

## AQL‑06 — Rule/axiom selectors

### AQL‑06.1 — By series

```aql
IS_AXIOM()
IS_RULE()
```

Examples:

```aql
SELECT * FROM ALL WHERE IS_AXIOM();

SELECT * FROM ALL WHERE IS_RULE();
```

### AQL‑06.2 — By namespace

```aql
NAMESPACE(prefix)
```

Example:

```aql
WHERE NAMESPACE("AX-")
```

---

## AQL‑07 — Options

### AQL‑07.1 — Sorting

```aql
OPTIONS ORDER BY field [ASC|DESC]
```

Example:

```aql
SELECT FIELDS(id, status)
FROM RULES
WHERE integrity = fail
OPTIONS ORDER BY id ASC;
```

### AQL‑07.2 — Limits

```aql
OPTIONS LIMIT n
```

Example:

```aql
SELECT * FROM NOTES
OPTIONS LIMIT 50;
```

## AQL‑08 — Predefined queries (canonical patterns)

These are **patterns**, not macros—intended for reuse.

### AQL‑08.1 — Find all failing integrity objects

```aql
SELECT FIELDS(id, type, location)
FROM ALL
WHERE integrity = fail;
```

### AQL‑08.2 — Find rules with dead dependencies

```aql
SELECT FIELDS(id, depends_on)
FROM RULES
WHERE INTEGRITY_FAIL() AND DEPENDS_ON(LN-12);
```

(Assuming LN‑12 is the “valid link targets only” rule.)

### AQL‑08.3 — Find cross‑layer violations (candidate)

```aql
SELECT FIELDS(id, layer, depends_on)
FROM ALL
WHERE layer = operator AND DEPENDS_ON(AX-*)
;
```

(Engine interprets `AX-*` as namespace pattern.)

### AQL‑08.4 — Find orphaned notes

```aql
SELECT FIELDS(id, location)
FROM NOTES
WHERE NOT HAS_INBOUND(*) AND NOT HAS_OUTBOUND(*);
```

## AQL‑09 — Invariants

AQL must:

- **never mutate** objects
- **never infer** semantics
- **never synthesize** content
- **only expose** structural relations and fields
- **remain deterministic** for a given snapshot
