# 🟥 3. How Field_pattern_signal is computed (operator‑grade)

The membrane evaluates three structural surfaces:

### 1. **Pattern‑of‑Presence**

A co‑occurrence matrix of fields across snapshots:

- which fields appear together
- which fields never co‑occur
- which fields are mutually exclusive

This forms the **presence motif**.

### 2. **Pattern‑of‑Shape**

A topology map of field structure:

- flat vs nested
- list vs map
- stable ordering vs free ordering
- depth signatures

This forms the **shape motif**.

### 3. **Pattern‑of‑Semantics**

A semantic clustering of fields:

- identifiers
- timestamps
- resource descriptors
- state clusters
- capability clusters

This forms the **semantic motif**.

Together, these produce the **Field_pattern_signal vector** — the entity’s structural fingerprint.
