# **v0.3 Pre‑Flight Drift Check — Formal State‑Machine (Operator Topology)**

## **State Set**

- **S0 — Start**
- **S1 — LoadSignals**
- **S2 — CompareManifests**
- **S3 — DetectConflict**
- **S4 — ClassifyConflict**
- **S5 — AttachLineage**
- **S6 — ApplyOverrideCode**
- **S7 — Resolve**
- **S8 — SyncAllowed**
- **S9 — SyncBlocked**
- **S10 — Abort**

---

## **Transition Logic (The Real Spine)**

### **S0 → S1 (LoadSignals)**

Enter when pre‑flight is invoked.  
Load:

- `L.manifest`
- `C.manifest`
- `T.tombstones`
- `E.env_sanity`

If `E.env_sanity = FAIL` → **S10 Abort**.

---

### **S1 → S2 (CompareManifests)**

Compare:

- hash equality
- mtime+size
- tombstone deltas

If identical → **S8 SyncAllowed**.

If different → **S3 DetectConflict**.

---

### **S2 → S3 (DetectConflict)**

Detect:

- unilateral drift
- bilateral drift
- deletion vs modification
- tombstone mismatch

If no conflict → **S8 SyncAllowed**.  
If conflict → **S4 ClassifyConflict**.

---

### **S3 → S4 (ClassifyConflict)**

Assign conflict class:

- **C1 — Benign Drift**
- **C2 — Local Wins**
- **C3 — Cloud Wins**
- **C4 — True Conflict**
- **C5 — Tombstone Ambiguity**

If C1/C2/C3 → **S7 Resolve**.  
If C4/C5 → **S5 AttachLineage**.

---

### **S4 → S5 (AttachLineage)**

Attach:

- conflict ancestry
- last‑known‑good
- operator ID
- timestamp
- drift vector

If lineage incomplete → **S10 Abort**.  
Else → **S6 ApplyOverrideCode**.

---

### **S5 → S6 (ApplyOverrideCode)**

Check for operator override:

- `O.NONE` → proceed to **S7 Resolve**
- `O.LOCAL_FORCE` → treat as C2
- `O.CLOUD_FORCE` → treat as C3
- `O.DEFER` → **S9 SyncBlocked**
- `O.ABORT` → **S10 Abort**

---

### **S6 → S7 (Resolve)**

Resolution path depends on class:

- C1 → fast‑forward
- C2 → local overwrite
- C3 → cloud overwrite
- C4 → operator‑guided merge
- C5 → tombstone arbitration

If resolution succeeds → **S8 SyncAllowed**.  
If resolution fails → **S9 SyncBlocked**.

---

### **Terminal States**

- **S8 SyncAllowed** → rsync/OneDrive may proceed
- **S9 SyncBlocked** → operator intervention required
- **S10 Abort** → environment or lineage unsafe

---

## **Topology Summary (The Operator View)**

The v0.3 machine is:

- **Deterministic** — no ambiguous transitions
- **Lineage‑aware** — conflicts carry ancestry
- **Override‑safe** — operator codes never poison history
- **Abort‑capable** — environment sanity is a hard gate
- **Sync‑gating** — only S8 permits automation
