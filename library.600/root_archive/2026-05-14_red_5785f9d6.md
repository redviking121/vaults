# 🟥 Takeaway

**Capability role is the membrane that declares _what the entity provides_ — the functional capability it contributes to the system.**  
It sits beneath **Operational mode** and completes the **Role markers** stack.

If Functional class is _what kind of thing it is_,

---

# 🟥 1. What Capability role actually encodes

Capability role expresses the **functional offering** of the entity — the capability it exposes to the system.

Examples:

- **network** — provides connectivity, routing, sockets
- **storage** — provides persistence, volumes, block devices
- **compute** — provides execution, CPU, workers
- **security** — provides authentication, policy, isolation
- **io** — provides device access, streams, interfaces

These are not identity fields.  
They are **functional promises**.

Capability role answers the question:  
**“What does this entity _enable_?”**

---

# 🟥 2. The three capability strata

Capability roles fall into three strata, each consumed by a different membrane:

- **Core capability** — fundamental system-level functions (network, storage, compute)
- **Auxiliary capability** — supportive functions (logging, metrics, monitoring)
- **Specialized capability** — domain-specific functions (crypto, ML, virtualization)

This forms the **capability signature** of the entity.

Functional class → category  
Operational mode → behavior  
Capability role → offering
