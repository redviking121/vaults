# 🟥 3. The struct fields (operator‑grade)

Your struct needs exactly one field:

- **scanner** — an `Arc` to your host‑first scanner engine

Why only one?

Because the scanner already contains:

- platform detection
- package enumeration
- file hashing
- service/process enumeration
- schema serialization

This is the **Single_Source_Capability** pattern.

The tool struct should _not_ contain:

- configuration
- state
- caching
- logging

Those belong to the scanner engine or the Engine runtime, not the tool.
