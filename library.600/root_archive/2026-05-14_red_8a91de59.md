# **AQL Runner CLI Wrapper — v1.0**

**Namespace:** `AQL‑CLI-*`  
**Status:** Authoritative  
**Owner:** Red Horne  
**Scope:** Command‑line interface for invoking the AQL Test Runner  
**Purpose:** Provide a stable, deterministic CLI surface for executing AQL Test Corpus queries, groups, and full suites.

---

## **AQL‑CLI‑01 — Invocation Model**

The wrapper exposes a **single binary**:

```
aql-runner
```

All commands follow the invariant form:

```
aql-runner <command> [options]
```

The wrapper never mutates substrate state.  
It only loads → executes → reports.

---

## **AQL‑CLI‑02 — Commands**

### **1. run-all**

Execute the entire AQL Test Corpus.

```
aql-runner run-all
```

**Behavior:**

- Loads full object graph
- Loads all queries
- Executes in deterministic order
- Emits canonical report
- Returns exit code 0 on full match, 1 on any diff

---

### **2. run-query `<id>`**

Execute a single query by its canonical ID.

```
aql-runner run-query AQL-TC-Q-17
```

**Behavior:**

- Loads object graph
- Loads only the specified query
- Executes and compares to Expected Output Matrix
- Emits result + diff (if any)

---

### **3. run-group `<namespace>`**

Execute all queries under a namespace prefix.

```
aql-runner run-group AQL-TC-Q
```

**Behavior:**

- Loads object graph
- Loads all queries matching prefix
- Executes in deterministic order
- Emits grouped report

---

### **4. list-queries**

Enumerate all available queries in deterministic order.

```
aql-runner list-queries
```

**Behavior:**

- Reads query manifest
- Prints canonical IDs only
- No execution

---

### **5. dump-graph**

Emit the loaded structural object graph for inspection.

```
aql-runner dump-graph
```

**Behavior:**

- Loads object graph
- Emits canonical JSON
- No execution

---

## **AQL‑CLI‑03 — Options**

### **`--format <fmt>`**

Output format: `text` (default), `json`, `yaml`.

```
aql-runner run-all --format json
```

### **`--trace`**

Emit full execution trace.

```
aql-runner run-query AQL-TC-Q-05 --trace
```

### **`--no-color`**

Disable ANSI color.

### **`--strict`**

Fail on warnings.

---

## **AQL‑CLI‑04 — Exit Codes**

|Code|Meaning|
|---|---|
|**0**|All results match expected outputs|
|**1**|One or more diffs detected|
|**2**|Invalid command or malformed input|
|**3**|Internal runner error (non‑deterministic behavior detected)|

---

## **AQL‑CLI‑05 — Wrapper Architecture**

The wrapper binds to the Test Runner via a fixed contract:

- **runner.load_graph()**
- **runner.load_queries(set)**
- **runner.execute()**
- **runner.compare()**
- **runner.report()**

The wrapper performs **no interpretation**.  
No mutation.  
No semantic inference.  
It is a pure I/O membrane.

---

## **AQL‑CLI‑06 — Determinism Guarantees**

The wrapper enforces:

- fixed ordering of queries
- fixed ordering of object graph load
- fixed output formatting
- no environment leakage
- no concurrency
- no nondeterministic iteration

If nondeterminism is detected, wrapper returns **exit code 3**.
