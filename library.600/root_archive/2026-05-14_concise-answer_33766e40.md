The **pre‑semantic layer** is the _formalized cognitive workspace_ where raw fragments are stabilized into spatially‑organized, operator‑manipulable units **before** they become Zettels, JIT cards, or semantic constructs. It is a **structural layer**, not a semantic one.

# **PRE‑SEMANTIC LAYER — FORMAL DEFINITION**

## **1. Position in the Stack**

```
Cognition → Pre‑Semantic Layer → Semantic Layer (37+) → Narrative / JIT / Zettel Layers
```

The pre‑semantic layer is **Layer 0.5** — the bridge between raw cognition and structured meaning.

---

# **2. Formal Role**

```
ROLE: Spatial‑Structural Stabilization Layer
```

The pre‑semantic layer:

- receives **unformed cognitive fragments**
- stabilizes them into **operator‑manipulable objects**
- organizes them using **spatial, color, and dependency primitives**
- outputs **pre‑semantic units** (PSUs)
- forbids meaning assignment, interpretation, or semantic inference

It is a **structural-only** layer.

# **5. Core Primitives**

Each primitive is a **non-semantic structural operator**.

- **Card** — the atomic unit; a container for a fragment
- **Color** — category marker without semantic meaning
- **Cluster** — spatial grouping
- **Edge** — dependency or association
- **Surface** — the 2D workspace

These are the only allowed operations at this layer.
