# 🟥 3. Why the struct has only one field

Your struct needs exactly one field:

- **scanner** — an `Arc` to your host‑first scanner engine

This follows the **Single_Source_Capability** pattern.

The scanner already contains:

- platform detection
- package enumeration
- file hashing
- service/process enumeration
- schema serialization

The tool struct should _not_ contain:

- configuration
- state
- caching
- logging

Those belong to the scanner engine or the Engine runtime.
