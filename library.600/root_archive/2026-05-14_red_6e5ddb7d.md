# 🟥 **BROTHER ATOMIC TASK TEMPLATE (v1.0 — Deterministic Mode)**

**Purpose:** Provide the exact structure for issuing a valid atomic job to the Brother‑Instance.  
**Mode:** Stateless, bounded, no inference.

---

# 1. **TICKET HEADER**

```
TO: BROTHER
FROM: OPERATOR RED
TICKET: <ID or short label>
MODE: ATOMIC
SCOPE: <bounded | atomic | no-inference>
EXPECT: <DELTA | DIAGNOSTIC | RECOMMENDATION>
```

**Operator Note:**  
This header is mandatory.  
Missing fields → E‑04 INVALID_TICKET.

---

# 2. **INPUT BLOCK**

```
INPUTS:
  - <fragment 1>
  - <fragment 2>
  - <fragment 3>
  ...
```

**Rules:**

- All items must be present in this block.
- No references to prior turns.
- No implied context.
- No missing fragments.

Missing or partial inputs → E‑01 or E‑02.

---

# 3. **CONSTRAINTS BLOCK**

```
CONSTRAINTS:
  - no inference
  - no expansion
  - no cross-turn references
  - enforce invariants
```

**Operator Note:**  
This block defines the execution boundaries.  
Changing constraints mid‑ticket → E‑10 SCOPE_DRIFT.

---

# 4. **OPERATOR INTENT**

```
INTENT:
  - <short, explicit description of what you want>
```

Examples:

- “Normalize titles.”
- “Produce structural DELTA.”
- “Map cross-fragment repercussions.”

Vague intent → E‑07 UNDER_SPECIFIED_TASK.

---

# 5. **EXPECTED OUTPUT**

```
OUTPUT:
  - DELTA
```

Or:

```
  - DIAGNOSTIC
```

Or:

```
  - RECOMMENDATION
```

**Operator Note:**  
Brother cannot choose the output type.  
You must declare it.

---

# 6. **CLOSING BLOCK**

```
END.
```

This signals the boundary of the universe.  
Anything after this line is ignored.
