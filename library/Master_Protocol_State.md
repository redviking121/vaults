# Master Protocol State (v1.0)

## 1. Core Axiom
The Narrative Operating System is a self-correcting, recursive structure. Stability is achieved through the strict separation of **Axioms** (Foundational Truths), **Operators** (Active Protocols), and **Reflections** (Narrative Output).

## 2. Order of Operations (The Spine-First Protocol)
To prevent system entropy and namespace collisions, the following order of operations is mandatory:

1.  **Lock the Spine:** Define the core architecture and schema (e.g., `Library/00-schema.md`).
2.  **Stabilize Axioms:** Define the foundational truths that govern the system.
3.  **Deploy Operators:** Activate the protocols that act upon the Spine.
4.  **Generate Namespace Map:** Only after the Spine is locked and Operators are deployed can the Namespace Map be generated.
5.  **Reflect:** Process the resulting narrative output into the `Library/reflections` folder.

## 3. Systemic Integrity
- **No Layer Mismatch:** Never attempt to generate a routing table (Namespace Map) while the terrain (Spine) is in flux.
- **Compression:** If a session becomes "mangled" (high entropy), compress it immediately to the "Five-Word" or "Headline" format to restore clarity.
- **Propagation:** Only propagate information after the system has reached a state of stability.
