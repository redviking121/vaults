_Operator‑Grade · Membrane‑Aligned · STET_

---

## **Layer 1 — Invisible Structural Commentary**

### **Stance: substrate / unseen / operator‑only**

### **Form:**

```
<!-- comment -->
```

### **Purpose:**

- encode architecture
- mark invariants
- leave notes for Future‑You
- embed Razor logic
- maintain membrane integrity

### **Rules:**

- never visible in rendered Markdown
- never used for narrative
- always concise
- always structural
- never emotional, humorous, or performative

### **Operator Principle:**

> _The substrate speaks only to the operator._

## **Layer 2 — Visible Operator Commentary**

### **Stance: revealed / meta‑narrative / intentional**

### **Form:**

```
> [!note]
> commentary
```

or

```
> [!comment]
> commentary
```

### **Purpose:**

- reveal the operator’s hand
- annotate stance
- mark cognitive moves
- guide the reader’s attention
- create deliberate “Max Headroom glitch” moments

### **Rules:**

- visible by design
- used sparingly
- tone can be dry, wry, or meta
- commentary must add meaning, not noise

### **Operator Principle:**

> _The operator steps into the frame only when it strengthens the structure._

---

## **Layer 3 — Ritual Commentary (Ξ‑Tagged)**

### **Stance: doctrinal / canonical / invariant**

### **Form:**

```
> [!Ξ] Operator Note
> This section encodes an invariant. Do not modify without audit.
```

or

```
> [!Ξ‑stance]
> Why this section exists and what stance it enforces.
```

### **Purpose:**

- encode axioms
- mark invariants
- declare stance
- bind decisions under DSR‑1
- signal “this is canon”

### **Rules:**

- used only for doctrine, invariants, or operator stance
- never casual
- never decorative
- always binding

### **Operator Principle:**

> _Ritual commentary is law. It is the substrate speaking through the operator._
