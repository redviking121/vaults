# **THE RECOVERY KERNEL — API SURFACE**

### _How higher layers call into the minimal executable core_

The Kernel exposes **three primitives** and **three invariants**.  
Nothing else.  
No hidden methods.  
No backdoors.  
No “advanced mode.”

Everything above v0.x must call through these.

---

## **I. The Three Primitives (the callable interface)**

These are the only _active_ calls higher layers can make.

### **1. `DETECT(deviation_signal)`**

**Purpose:** Feed the Kernel a signal that _something is off_.  
Higher layers do not diagnose — they _report_.

**Input:**

- Any deviation vector: cognitive, emotional, physiological, operational, environmental.
- Raw, uncompressed, unfiltered.

**Output:**

- A normalized deviation metric.
- A flag: `stable`, `drifting`, or `collapsing`.

**Guarantee:**  
The Kernel never ignores a deviation.  
It always updates its internal state.

---

### **2. `HALT(collapse_vector)`**

**Purpose:** Request immediate stabilization when collapse is detected or predicted.

**Input:**

- A collapse vector (direction + magnitude).
- Optional urgency hint from higher layers.

**Output:**

- A stabilization directive (what must stop _right now_).
- A temporary override of higher‑layer processes if needed.

**Guarantee:**  
The Kernel will always choose the _minimum viable intervention_ that prevents collapse.  
Never more.  
Never less.

---

### **3. `RESTORE(minimum_coherence_spec)`**

**Purpose:** Ask the Kernel to bring the system back to a baseline where higher layers can resume.

**Input:**

- A spec describing the minimum viable coherence required for the next layer to function.  
    (e.g., “restore working memory to 30%,” “restore emotional bandwidth to safe range,” “restore operational clarity to threshold X.”)

**Output:**

- A coherence restoration plan.
- A readiness flag: `not_ready`, `booting`, `ready`.

**Guarantee:**  
The Kernel restores _only_ the minimum coherence required to hand control back upward.

---

## **II. The Three Invariants (the non‑callable contract)**

Higher layers cannot call these.  
They are _guarantees_ the Kernel enforces on itself.

### **Invariant 1 — “Never Lie About State”**

The Kernel will always return the true stability state, even if it is catastrophic.

### **Invariant 2 — “Never Allow Collapse to Continue Unchecked”**

If collapse is detected, the Kernel _must_ intervene.  
It cannot be overridden.

### **Invariant 3 — “Always Restore a Minimum Viable Self”**

The Kernel will always bring the system back to a state where higher layers can resume execution.

These invariants are not functions.  
They are **laws**.

---

## **III. The Call Discipline (how higher layers must behave)**

Higher layers must follow three rules when calling the Kernel:

### **Rule 1 — No Compression Before Detection**

Higher layers must not pre‑interpret deviation.  
They must send raw signals.

### **Rule 2 — No Negotiation During Collapse**

Once `HALT()` is invoked, higher layers must yield.  
No bargaining.  
No “just one more task.”

### **Rule 3 — No Demands Beyond Minimum Coherence**

Higher layers may request _minimum viable_ coherence only.  
Anything more is a violation.

---

## **IV. The Kernel’s Return Protocol (what higher layers receive)**

Every Kernel call returns a **three‑part packet**:

1. **State:** `stable`, `drifting`, `collapsing`, `recovering`, `ready`
2. **Directive:** what the higher layer must do or stop doing
3. **Time Horizon:** how long the directive is valid

This ensures higher layers never guess.

---

## **V. The Meta‑Rule: The Kernel Cannot Be Bypassed**

No higher layer — not memory, not agency, not will, not purpose, not platform — may bypass the Kernel.

If they try, the Kernel treats it as a deviation.

This is how the architecture prevents self‑destruction.
