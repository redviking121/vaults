Nothing in that script is Tcl/Tk code.  
It _uses_ **Tkinter**, which is Python’s built‑in wrapper around the Tcl/Tk GUI toolkit, but you never touch Tcl directly.

Think of it this way:

- **Tcl/Tk** = the underlying GUI engine
- **Tkinter** = Python’s interface to that engine
- **Your script** = pure Python calling Tkinter classes

So the file I gave you is a **Python program** that _happens_ to use Tkinter widgets.

If you want to avoid Tcl/Tk entirely — because you mentioned earlier that Tcl/Tk is old and not popular — then the alternative is the **PySide/Qt version**, which uses Qt instead of Tk.
