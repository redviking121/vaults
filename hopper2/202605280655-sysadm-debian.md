That "waiting for `/dev/root`" hang during a NixOS boot usually means the Linux kernel fired up just fine, but the stage 1 boot process is stuck blindly groping around in the dark, unable to find or mount the actual installation media (often due to USB 3.0 controller quirks or missing storage drivers in the initramfs).

Since you have a fully functional, sacrificial Debian 13 installation sitting right next to your failsafe drive, you absolutely can install NixOS directly from inside Debian. In fact, it's often much easier on quirky hardware because you get to bypass the flaky hardware-detection phase of the standard live installer ISO.

The cleanest, most reliable way to pull this off is by using **nixos-anywhere** or the traditional **nix-user-chroot** method. Here is the straightforward blueprint to turn that target Debian drive into your new NixOS environment without touching a USB stick.

## The In-Place Installation Roadmap

### 1. Install Nix inside Debian

First, you need the Nix package manager running on your target Debian system so you can access the NixOS installation tools. Open your Debian terminal and run the official multi-user installer:

Bash

```
curl -L https://nixos.org/nix/install | sh
```

Follow the prompts, restart your shell as instructed to source the new environment variables, and verify it works by checking the version: `nix-env --version`.

### 2. Format the Target Partition(s)

You need to prep your target drive. Since you are already running Debian on it, you can shrink or wipe the specific partitions you intend to give to NixOS.

- Make sure you have a **boot/ESP partition** (usually vfat/FAT32 formatted to `/boot` or `/boot/efi`).
    
- Format your main root partition to `ext4` (or `btrfs` if that's your flavor).
    

> ⚠️ **Double-check your drives:** Use `lsblk` to ensure you are modifying the target drive and not your failsafe drive.

Mount the newly formatted partitions under a temporary directory like `/mnt`:

Bash

```
sudo mkdir -p /mnt
sudo mount /dev/sdX2 /mnt  # Your target root partition
sudo mkdir -p /mnt/boot
sudo mount /dev/sdX1 /mnt/boot  # Your target EFI partition
```

### 3. Generate Your NixOS Configuration

Now, use the Nix tools you installed in step 1 to generate a fresh hardware configuration based on what Debian sees. We will run `nixos-generate-config` out of a temporary Nix shell:

Bash

```
nix-shell -p nixos-install-tools --run "sudo -E nixos-generate-config --root /mnt"
```

This drops two critical files into `/mnt/etc/nixos/`:

- `hardware-configuration.nix`: This detects the exact driver modules your Panasonic CF-53 needs to mount the root drive (which should bypass the `/dev/root` bug entirely!).
    
- `configuration.nix`: Your main system config.
    

### 4. Edit Your Configuration

Open `/mnt/etc/nixos/configuration.nix` in your favorite editor. You **must** ensure the bootloader settings are correct for your CF-53 architecture (likely UEFI):

Nix

```
boot.loader.systemd-boot.enable = true;
boot.loader.efi.canTouchEfiVariables = true;
```

_(If your Toughbook is configured for legacy MBR boot instead of UEFI, you'll need to configure `boot.loader.grub` targeting the specific drive instead)._

### 5. Fire the Installation Trigger

With the target filesystem mounted at `/mnt` and your config files generated, run the installer from your Debian host:

Bash

```
nix-shell -p nixos-install-tools --run "sudo -E nixos-install --root /mnt"
```

This will pull down all the necessary binaries, build your NixOS system closure, install it directly onto the target partition, and register NixOS with the system's boot variables.

Once it completes successfully, set a root password when prompted, type `sudo reboot`, and your CF-53 should boot straight into a clean NixOS environment, completely ignoring whatever poltergeist was haunting your USB media ports.
