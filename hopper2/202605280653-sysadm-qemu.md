Running QEMU on NixOS—or running NixOS inside QEMU—is a match made in heaven. Because NixOS is entirely declarative, it eliminates the usual headache of managing complex QEMU command-line arguments or drift in your VM configurations.

There are two primary ways you might be looking to combine these tools:

1. **Using NixOS as the Host** to run and manage QEMU virtual machines.
    
2. **Using QEMU to test NixOS configurations** safely inside a lightweight VM.
    

## 1. NixOS as the Host (Running QEMU/KVM)

To get full KVM hardware acceleration and a clean management interface on a NixOS host, you don't manually install packages and configure network bridges. Instead, you declare them in your `/etc/nixos/configuration.nix`.

Add the following snippets to enable the `libvirtd` daemon (which manages QEMU) and install **Virt-Manager** for a graphical interface:

Nix

```
# /etc/nixos/configuration.nix
{ config, pkgs, ... }: {

  # Enable libvirtd system daemon for QEMU/KVM
  virtualisation.libvirtd.enable = true;

  # Install Virt-Manager graphical interface
  programs.virt-manager.enable = true;

  # Add your user account to the libvirtd group to manage VMs without sudo
  users.users.YOUR_USERNAME.extraGroups = [ "libvirtd" ];
}
```

After updating your configuration, simply run `sudo nixos-rebuild switch` and log out/log back in to apply the group changes. You can then launch `virt-manager` from your application menu or run native `qemu-kvm` commands from the terminal.

## 2. Testing NixOS via QEMU (The Declarative Way)

One of NixOS's best superpowers is its built-in VM generator. If you are experimenting with a new system configuration, a risky network setup, or a new desktop environment, you can test it in QEMU instantly **without downloading an ISO file** or modifying your real hardware.

### Option A: The Instant Test Drive (Using Nix Flakes)

If you have Flakes enabled, you can spin up a fully functional, isolated QEMU virtual machine running a stock NixOS environment with a single terminal command:

Bash

```
nix run nixpkgs#nixosConfigurations.vm.config.system.build.vm
```

This builds the operating system in your local Nix store, creates a virtual disk image in your current directory, and launches QEMU with KVM acceleration automatically.

### Option B: Building a VM from a Custom Configuration

If you want to test a specific configuration file (for example, a `vm.nix` file with experimental settings), you can define a micro-configuration and have NixOS generate a tailored QEMU launch script for it.

Here is a minimalist template for an experimental VM configuration:

Nix

```
# vm.nix
{ pkgs, ... }: {
  # Inherit standard guest agent settings for QEMU
  virtualisation.vmVariant = {
    # Configure the virtual hardware assigned to the QEMU instance
    virtualisation = {
      memorySize = 4096; # Allocate 4GB RAM
      cores = 4;         # Allocate 4 CPU cores
      graphics = true;   # Enable standard QEMU display window
    };
  };

  # Automatically log in as a test user
  services.getty.autologinUser = "alice";
  users.users.alice = {
    isNormalUser = true;
    extraGroups = [ "wheel" ]; # Give sudo privileges inside the VM
    password = "password";
  };

  # Add any experimental packages you want to test
  environment.systemPackages = with pkgs; [
    git
    htop
  ];
}
```

To build and launch this specific configuration in QEMU, run:

Bash

```
# 1. Build the system configuration and generate the QEMU wrapper script
nix-build <nixpkgs/nixos> -A vm -I nixos-config=vm.nix

# 2. Run the generated script to launch QEMU
./result/bin/run-nixos-vm
```

When the QEMU window opens, any changes you make or files you save inside the VM will persist in a small `.qcow2` disk file created in that directory, leaving your host system completely untouched.
