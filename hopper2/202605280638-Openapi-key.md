Setting up your `OPENAI_API_KEY` as an environment variable keeps your credentials secure and prevents you from accidentally leaking secrets in your source code.

Here is how to set it up across different operating systems, followed by how long a key remains valid.

## How to Set the Environment Variable

### 1. macOS and Linux

To make the variable persistent across all terminal sessions, add it directly to your shell profile configuration file (usually `.zshrc` or `.bashrc`).

1. Open your terminal.
    
2. Run the following command to append the variable to your profile (replace `your_actual_api_key_here` with your secret key):
    
    Bash
    
    ```
    echo 'export OPENAI_API_KEY="your_actual_api_key_here"' >> ~/.zshrc
    ```
    
    _(Note: If you use Bash instead of Zsh, replace `~/.zshrc` with `~/.bashrc` or `~/.bash_profile`.)_
    
3. Refresh your current terminal window to apply the changes:
    
    Bash
    
    ```
    source ~/.zshrc
    ```
    
4. Verify it was set correctly by printing it:
    
    Bash
    
    ```
    echo $OPENAI_API_KEY
    ```
    

### 2. Windows

You can set this via the command line or through the graphical user interface.

#### Option A: Command Prompt (Quickest)

1. Open the **Command Prompt**.
    
2. Run the `setx` command (this sets a permanent user environment variable):
    
    DOS
    
    ```
    setx OPENAI_API_KEY "your_actual_api_key_here"
    ```
    
3. **Important:** Close that Command Prompt window and open a _new_ one for the changes to take effect. You can verify it by running:
    
    DOS
    
    ```
    echo %OPENAI_API_KEY%
    ```
    

#### Option B: Windows GUI

1. Press the Windows Key and search for **"Edit the system environment variables"**.
    
2. Click the **Environment Variables...** button at the bottom right.
    
3. Under the **User variables** section (the top half), click **New...**.
    
4. Set the **Variable name** to `OPENAI_API_KEY` and the **Variable value** to your actual secret key string.
    
5. Click **OK** on all windows to save. Restart your IDE or terminal window to ensure it reads the new variable.
    

## How Long is an API Key Viable?

OpenAI API keys **do not have a built-in expiration date**. Once generated, a key remains viable indefinitely under normal conditions.

However, a key will stop working immediately if any of the following occur:

- **Manual Deletion:** If you click the trash icon next to the key in your OpenAI developer dashboard, it is instantly revoked.
    
- **Leaked Key Detection:** OpenAI actively scans public repositories (like GitHub). If their automated systems detect that your key has been committed to a public repository, they will automatically revoke it for safety and notify you via email.
    
- **Credit Expiration / Account Delinquency:** While the _key itself_ doesn't expire, the funds attached to it can.
    
    - **Paid Credits:** Pre-funded balance you purchase typically expires **1 year** from the date of purchase.
        
    - **Trial Credits:** Free promotional credits given to new accounts usually expire after **3 months**.
        
    - If your account balance hits $0 or expires, any API key associated with that account will return a `429 (You exceeded your current quota)` error until you top up your balance.
