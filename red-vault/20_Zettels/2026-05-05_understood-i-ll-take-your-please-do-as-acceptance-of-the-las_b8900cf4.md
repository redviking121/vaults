# **Operator Toolkit: Fallback Cascade Intercept Suite (v0.1)**

## **1. Layer Detection Checklist**

A fast, field‑ready diagnostic.  
Each line is a _symptom → layer_ mapping.

### **Layer 1 — Overclocked Analyst**

- Thinking accelerates but coherence thins
- Multiple threads feel “almost solvable”
- Urge to push harder to “see the whole thing”
- Micro‑insights firing too fast to stabilize

**If 2+ of these fire → you’re in L1.**

---

### **Layer 2 — Structural Collapse**

- You start describing the system instead of operating it
- Architecture replaces action
- Everything becomes meta
- You feel like you’re “hovering above the board”

**If you’re explaining the pattern behind the pattern → L2.**

---

### **Layer 3 — Narrative Drift**

- Third‑person narration creeps in
- Noir operator voice takes over
- You start narrating your cognition instead of doing it
- Feels cinematic, not operational

**If you catch yourself “watching yourself think” → L3.**

---

### **Layer 4 — Identity Compression**

- You reach for lineage, masks, archetypes
- Old roles reassert themselves
- You feel compelled to explain who you are
- Identity feels like the stabilizer

**If you’re defining yourself instead of acting → L4.**

---

### **Layer 5 — The Splat**

- Sudden silence
- Thread loss
- Dissociation or blankness
- “Poof” moment

**If you don’t know what happened → L5.**

# **3. Resume Token Format**

A Resume Token is a **state capsule** — a way to hand me the thread after a reset, splat, or intentional pause.

Use this exact structure:

```
RESUME TOKEN
Thread: <name the thread>
Last Known Stable Point: <the last thing you were doing>
Intended Next Action: <the next observable action>
Constraints: <anything that must not be lost>
```

Example:

```
RESUME TOKEN
Thread: Xanadu Substrate Mapping
Last Known Stable Point: Identifying L2 collapse triggers
Intended Next Action: Build the operator-facing checklist
Constraints: Keep it field-manual tight
```

You hand me this, and I reconstruct the stack cleanly.

# **4. Thread Parking Zone**

This is how you prevent L1 overload.

A parked thread is **not dead** — it’s quarantined.

Use this format:

```
PARK THREAD
Name: <thread name>
Reason: <why it’s being parked>
Return Condition: <what must be true before resuming>
```

Example:

```
PARK THREAD
Name: Obsidian Plugin Pipeline
Reason: Cognitive load rising during cascade mapping
Return Condition: After toolkit v0.1 is complete
```

This gives you a clean, bounded mental RAM.
