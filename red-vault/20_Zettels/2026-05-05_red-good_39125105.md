# **LOG INGESTION PROTOCOL v1.0 — Operator Grade**

A routing protocol tells you _where logs go_.  
An ingestion protocol tells you _how they get there_.

This is the part that makes the Academy self‑documenting.

---

## **1. Ingestion Sources**

Every log in your environment comes from one of three emitters:

- **Terminal** — shells, commands, operator actions
- **WebUI** — model events, session metadata, agent actions
- **Systemd** — services, failures, boots, restarts

Each source has a different cadence and semantic weight, but they all feed the same ingestion layer.

---

## **2. Ingestion Contract**

Every log entering the Academy must satisfy:

```
{
  timestamp: ISO8601,
  source: "terminal" | "webui" | "systemd",
  session: SessionID | null,
  level: "info" | "warn" | "error",
  message: string,
  metadata: object
}
```

This is the **canonical shape**.  
Everything else is normalization.

---

## **3. Terminal Ingestion**

Terminal logs are the most chaotic, so they get the strongest normalization.

### **Capture Methods**

- shell `PROMPT_COMMAND` hook
- `script`-style session recorder
- wrapper function around your shell (`xi-sh`)
- or a simple `tee` pipe for low‑friction capture

### **Normalization**

- strip ANSI
- collapse multi‑line commands
- detect command vs output
- attach session ID if inside a Session

### **Routing**

Terminal logs → **Channel 1 (Operator)** by default.

---

## **4. WebUI Ingestion**

This is the cleanest source.

### **Captured Automatically**

- model calls
- agent actions
- session creation
- session switches
- tool invocations
- errors

### **Normalization**

- flatten nested metadata
- attach SessionID
- classify event type

### **Routing**

WebUI logs → **Channel 2 (System)** by default.

---

## **5. Systemd Ingestion**

This is the heartbeat of the machine.

### **Captured via**

- `journalctl -f` tailer
- systemd unit `StandardOutput=journal`
- optional sidecar service

### **Normalization**

- extract unit name
- detect severity
- attach boot ID

### **Routing**

Systemd logs → **Channel 3 (Machine)** by default.

---

## **6. Ingestion Flow**

All three sources feed the same pipeline:

```
Capture → Normalize → Classify → Route → Store
```

### **Capture**

Raw text, events, or journal entries.

### **Normalize**

Convert to the ingestion contract.

### **Classify**

Determine:

- channel
- severity
- session
- operator relevance

### **Route**

Send to the correct log channel.

### **Store**

Append to the Academy’s log ledger.

---

## **7. Ingestion Guarantees**

The ingestion layer must guarantee:

- **No loss** — logs never silently disappear
- **Order preservation** — timestamps remain monotonic
- **Session coherence** — logs stay bound to their vector
- **Replayability** — you can reconstruct any session from logs alone

This is what makes the Academy _auditable_.
