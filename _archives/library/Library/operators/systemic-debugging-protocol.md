# Systemic Debugging Protocol

This protocol defines the approach for treating technical friction as a diagnostic signal to maintain operational sovereignty.

## Workflow
1. **Identify:** Recognize the point of friction (e.g., font scaling, window management, sandbox restrictions).
2. **Isolate:** Determine if the issue is local to the application or a constraint imposed by the substrate (Wayland, X11, OS).
3. **Analyze:** Map the "Trifecta of Failure" (Electron + Wayland + Sandbox) to understand the architectural reason for the resistance.
4. **Bypass/Resolve:** Implement a forensic workaround that restores operational clarity without compromising the system's integrity.
5. **Document:** Record the diagnostic signal in the Archive to inform future system architecture.
