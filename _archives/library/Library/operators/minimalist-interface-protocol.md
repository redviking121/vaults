# Minimalist Interface Protocol

This protocol defines the approach to tool selection and usage, prioritizing interfaces that are quiet, unobtrusive, and pedagogical.

## Core Principle
**Interface-as-Pedagogy.** A tool that hides its functionality is not a design flaw; it is a teacher. By removing visual cues, the tool forces the Operator to internalize the workflow, eventually making the interface disappear.

## Workflow
1. **Selection:** Prioritize tools that are minimal and require manual configuration (e.g., Sublime Text, CLI tools).
2. **Internalization:** Treat the "lack of features" or "hidden menus" as a challenge to learn the underlying logic of the tool.
3. **Transparency:** Once the workflow is internalized, the tool becomes an extension of the Operator's thought process.
4. **Integration:** Link the creative session note to this protocol to maintain the context of the tool's adoption.
