# Vault Topology Protocol

This protocol defines the vault as a tri-layer reactive knowledge substrate, balancing the structural skeleton, the neural Zettels, and the JIT operational templates.

## Core Principle
**The vault is a living organism.** Its structure must emerge from the Operator's thinking, not be imposed upon it.

## The Three-Mask Topology
1. **Systems/Substrate:** The foundational infrastructure (NixOS, protocols, hardware).
2. **Comms/PR:** The outward-facing interface (Presence-building, alignment).
3. **Intelligence/Operator:** The core cognitive engine (Zettels, axioms, reflections).

## Workflow
1. **Observe:** Monitor the vault's natural growth patterns.
2. **Align:** Ensure the topology supports the three-mask balance.
3. **Refine:** Prune legacy cruft to maintain reactivity.
