# Presence Building Protocol

## Purpose
To establish an authentic, sustainable presence within technical ecosystems by leveraging existing technical and philosophical alignment.

## Workflow
1. **Identify Alignment:** Map your existing work (e.g., RISC-V, NixOS, Open Hardware) to the core values of the target ecosystem.
2. **Engage as Native:** Participate in the community's technical discourse. Do not "market"—contribute.
3. **Demonstrate Competence:** Solve real problems within the ecosystem.
4. **Maintain Presence:** Presence is a continuous state of alignment, not a campaign.
