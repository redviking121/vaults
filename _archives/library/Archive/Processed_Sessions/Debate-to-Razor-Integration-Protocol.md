# Debate-to-Razor Integration Protocol

## Purpose
To formalize the **Debate Engine Protocol** as the primary mechanism for **Contradiction Resolution** within the **Razor Governance** and **TRMS Relational Dynamics** frameworks.

## Integration Logic
The Debate Engine acts as the "Dialectic Vessel" for resolving conflicts identified by the Razor. When a state of contradiction arises between frames or agents, the system triggers the Debate Engine to move from *tension* to *decision stability*.

## Operational Flow
1. **Detection:** A contradiction is flagged within the [[202605280957-razor-governance-map]] or [[202605280959-trms-map]].
2. **Initialization:** The system invokes the [[2026-05-14_debate-engine-protocol_aa4fd2da]].
3. **Dialectic Execution:**
    - **Proponent:** Argues for the "Upshift" (coherence/expansion).
    - **Opponent:** Argues for the "Downshift" (stability/pruning).
4. **Verdict & Synthesis:** The Moderator/Judge produces a verdict that updates the system state, ensuring **Propagation Coherence**.
