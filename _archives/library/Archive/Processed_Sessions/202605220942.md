## ⭐ Path B — The “System‑Level” Fix (XWayland fallback)

Electron apps behave better under XWayland than native Wayland on KDE.

You can force Copilot to run under XWayland:

```
env QT_QPA_PLATFORM=xcb copilot-desktop
```

This makes KDE’s scaling rules work again.

- **Run under XWayland**

## ⭐ Why the window‑rules detection failed

You hit the trifecta:

- Copilot is an **Electron** app
- Running under **Wayland**
- Inside a **sandbox/container**

Wayland deliberately blocks cross‑window metadata access for security.  
So KDE’s “Detect Window Properties” can’t see the app.

That’s why no crosshair, no communication, no rules.
