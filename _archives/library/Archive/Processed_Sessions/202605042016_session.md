Easiest path: **you generate a plain JPG of a single color using either ImageMagick or GIMP**.  
Since you’re clearly operating in an Ubuntu‑ish operator stance, here are the clean, reversible ways to do it.

---

## 🧱 Option 1: Generate a solid‑color JPG via ImageMagick (fastest, no ceremony)

If you have ImageMagick installed:

```bash
sudo apt install imagemagick
```

Then create a 4K solid‑color wallpaper (example: pure black):

```bash
convert -size 3840x2160 xc:"#000000" wallpaper.jpg
```

Or a soft gray:

```bash
convert -size 3840x2160 xc:"#2e2e2e" wallpaper.jpg
```

Or any hex color you want:

```bash
convert -size 3840x2160 xc:"#RRGGBB" wallpaper.jpg
```

Then drop it into the system wallpaper directory:

```bash
sudo cp wallpaper.jpg /usr/share/backgrounds/
```

That’s it — it will show up in the GNOME wallpaper picker.

---

## 🎨 Option 2: Use GIMP (GUI, if you want to eyeball the color)

1. Open GIMP
2. File → New → set resolution (e.g., 3840×2160)
3. Bucket Fill → choose your color
4. Export As → `wallpaper.jpg`
5. Copy to `/usr/share/backgrounds/` with sudo

---

## 🧩 Option 3: Use a tiny Python one‑liner (if you like minimal dependencies)

```bash
python3 - << 'EOF'
from PIL import Image
img = Image.new("RGB", (3840, 2160), "#1a1a1a")
img.save("wallpaper.jpg", "JPEG")
EOF
```

Then copy it into place.
