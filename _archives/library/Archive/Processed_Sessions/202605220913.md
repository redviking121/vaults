Ah, yeah — Sublime can feel a little _mysterious_ the first time you try to save because it doesn’t shove a giant “SAVE ME NOW” button in your face. Once you know where it lives, though, it becomes second nature.

Here’s the clean, operator‑grade breakdown of **how saving works in Sublime**, and why it felt unclear at first.

---

## 🟦 The Sublime Save Path (the part that wasn’t obvious)

### **1. First save = “Save As…”**

When the file is brand new (like your `scan_missing_nodes.py`), Sublime doesn’t know where it belongs yet.  
So the first save is always:

**File → Save**  
or  
**File → Save As…**

Both open the same dialog the first time.

After that, **File → Save** just overwrites normally.

---

## 🟩 The actual menu path

Here’s the exact sequence:

- Click **File** in the top menu bar
- Choose **Save** (or **Save As…**)
- Pick the directory
- Name the file
- Hit **Save**

That’s it.

Sublime doesn’t auto‑prompt you because it assumes you might want to paste, edit, or inspect before committing.

---

## 🟥 Why it felt unclear

Because Sublime:

- doesn’t show a toolbar by default
- doesn’t show a “save” icon
- doesn’t pop up a save dialog automatically
- doesn’t warn you until you try to close the tab
- doesn’t auto‑save new files unless configured

It’s intentionally minimal.

If you’re used to editors that shove a save dialog at you immediately, Sublime feels like it’s hiding something.

It isn’t — it’s just quiet.

---

## 🟦 Optional: Add a visible save button

If you want a toolbar with icons (including a save icon), you can enable:

**View → Show Menu**  
**View → Show Tabs**  
**View → Show Status Bar**

Sublime doesn’t have a native toolbar with icons, but the menu bar is always available.

---

## 🟩 Operator‑grade shortcut

Once the file exists:

- **Ctrl+S** (Linux/Windows)
- **Cmd+S** (Mac)

Instant save. No dialog.
