## **What happened when you ran it from `.bashrc`**

Your tuned alias did three critical things:

1. **cd ~/dev/open-webui**  
    → Now the backend can _see_ the `build/` directory.
    
2. **source venv/bin/activate**  
    → Now the correct Python environment is active.
    
3. **open-webui serve**  
    → Now the correct entry point is running.
    

Those three together are what make the UI appear.

If any one of them is missing, you get:

```
{"detail":"Not Found"}
```

because the backend has no route to the frontend.
