# ⭐ **Most stage‑1 boot failures come from:**

**(1) missing or wrong root device**,  
**(2) missing kernel modules**,  
**(3) an initrd that doesn’t match the hardware**, or  
**(4) a filesystem that changed underneath the config.**

Stage‑1 is tiny, brittle, and declarative — so when it breaks, it breaks loudly.

---

# 🟥 **1. Wrong or changed root device**

This is the most common.

If the Toughbook’s drive enumeration changed (e.g., `/dev/sda` became `/dev/sdb`), stage‑1 can’t mount the root FS and dies.

**Symptoms:**

- “cannot mount /dev/disk/by‑uuid/…”
- “waiting for device…”
- “gave up waiting for root device”

**Fix:**  
Boot into the installer → `lsblk` → confirm the UUID → update `fileSystems."/"` in `configuration.nix` or `hardware-configuration.nix`.

---

# 🟥 **2. Missing kernel modules in initrd**

If your hardware needs a module that wasn’t included in the initrd, stage‑1 can’t bring up the root FS.

Common culprits on older Toughbooks:

- SATA/AHCI modules
- old Intel Wi‑Fi/PCI bridges
- USB storage drivers (if booting from USB)

**Fix:**  
Add to `boot.initrd.availableKernelModules`:

```nix
boot.initrd.availableKernelModules = [
  "ahci"
  "sd_mod"
  "sr_mod"
  "usb_storage"
  "usbhid"
];
```

Rebuild.

---

# 🟥 **3. Initrd built for a different kernel than the one installed**

This happens if a rebuild was interrupted or a generation mismatch occurred.

**Symptoms:**

- “kernel module not found”
- “module xyz not found in directory …”

**Fix:**  
Boot from installer → mount root → `nixos-rebuild switch --root /mnt` to force a clean rebuild.

---

# 🟥 **4. Filesystem type mismatch**

If you changed the FS (ext4 → btrfs, or vice‑versa) but didn’t update the config, stage‑1 can’t mount it.

**Fix:**  
Check `fileSystems."/".fsType` matches reality.

---

# 🟥 **5. LUKS or encrypted root not opening**

If you enabled encryption or inherited a config that expects it, stage‑1 will fail before mounting.

**Symptoms:**

- “no such device”
- “failed to open luks device”

**Fix:**  
Ensure:

```nix
boot.initrd.luks.devices.<name>.device = "/dev/disk/by-uuid/...";
```

---

# 🟥 **6. Hardware-configuration.nix drift**

If you copied configs between machines or drives, the auto‑generated hardware file may not match the Toughbook.

**Fix:**  
Boot installer → run:

```
nixos-generate-config --root /mnt
```

Compare the new hardware config to your existing one.
