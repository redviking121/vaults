# **PART 3 — THE HEADLINE DRILL**

**Formal Specification (v1.0)**  
_Defines the Operator’s compression protocol for collapsing complexity into a single, stable cognitive anchor._

---

# **0. PURPOSE**

Establish a deterministic method for reducing any situation, problem, or information set into a **single controlling question or statement** (“the headline”).  
The headline functions as the Operator’s **primary orientation vector**, preventing drift, overwhelm, or fragmentation.

---

# **1. SCOPE**

This module governs:

- cognitive compression
- situational framing
- mission alignment
- communication clarity
- decision‑making under load

It applies to:

- internal reasoning
- external communication
- mission planning
- threat assessment
- debriefs and after‑action reviews

---

# **2. DEFINITIONS**

**2.1 HEADLINE**  
The minimal, irreducible question or statement that captures the **core of the situation**.  
It must be singular, unambiguous, and load‑bearing.

**2.2 SIGNAL VECTOR**  
The directional intent encoded in the headline.  
Determines what the Operator pays attention to and what is ignored.

**2.3 COMPRESSION WINDOW**  
The time‑bounded interval in which the Operator must collapse complexity into the headline.  
Default: **≤ 30 seconds**.

**2.4 DRIFT**  
Any deviation from the headline’s signal vector.  
Includes: tangents, emotional contamination, ego‑reactivity, or unbounded exploration.

---

# **3. INVARIANTS**

The following conditions must always hold:

- A headline must be **singular** (one question or one statement).
- A headline must be **load‑bearing** (it must meaningfully constrain action).
- A headline must be **stable** (it does not mutate under pressure).
- A headline must be **minimal** (no excess language, no qualifiers).
- A headline must be **operational** (it must guide behavior).

If any invariant fails, the headline is invalid.

---

# **4. INPUT REQUIREMENTS**

The Operator must supply:

- the raw situation
- the mission context
- the desired outcome or constraint
- any known threat vectors
- any time or resource limitations

No additional data is required.

---

# **5. OUTPUT REQUIREMENTS**

The system must produce:

- **one** headline
- **one** signal vector
- optional secondary clarifiers (≤ 2), only if required for execution

Outputs must be deterministic and reproducible.

---

# **6. PROCESS SPECIFICATION**

**6.1 INITIALIZATION**  
The Operator invokes the HEADLINE Drill when:

- complexity exceeds working memory
- drift risk is detected
- mission clarity degrades
- communication requires compression
- decision latency increases

**6.2 EXTRACTION PHASE**  
The Operator identifies:

- the core tension
- the core unknown
- the core objective

All non‑core elements are discarded.

**6.3 COMPRESSION PHASE**  
The Operator collapses the extracted elements into a single headline using the following rules:

- Rule 1: Prefer **questions** over statements when uncertainty is high.
- Rule 2: Prefer **statements** over questions when action is required.
- Rule 3: Remove all adjectives, qualifiers, and emotional language.
- Rule 4: Reduce to **≤ 12 words** unless mission constraints require more.

**6.4 VALIDATION PHASE**  
The headline is tested against invariants (Section 3).  
If any invariant fails, return to 6.2.

**6.5 LOCK PHASE**  
Once validated, the headline becomes the **controlling frame**.  
All subsequent reasoning must reference it.

---

# **7. OPERATOR CONSTRAINTS**

- The Operator may not generate multiple headlines for a single situation.
- The Operator may not revise the headline without new information.
- The Operator may not proceed to planning or execution without a locked headline.
- The Operator must terminate any drift by re‑anchoring to the headline.

---

# **8. FAILURE MODES**

**8.1 OVER‑COMPRESSION**  
Headline becomes too narrow, excluding critical signal.  
Mitigation: Re‑expand extraction phase.

**8.2 UNDER‑COMPRESSION**  
Headline remains vague or multi‑part.  
Mitigation: Apply Rule 4 (≤ 12 words).

**8.3 EMOTIONAL CONTAMINATION**  
Headline reflects mood, not mission.  
Mitigation: Strip affective language.

**8.4 EGO‑REACTIVITY**  
Headline centers identity rather than objective reality.  
Mitigation: Re‑anchor to mission context.

---

# **9. SUCCESS CRITERIA**

A valid headline must:

- reduce cognitive load
- increase clarity
- constrain action
- eliminate drift
- accelerate decision‑making
- align the Operator with mission intent

If these conditions are met, the HEADLINE Drill is successful.
