# Why this works

A poisoned FUSE mountpoint is not a “mount” — it’s a directory whose inode is still tied to a dead FUSE superblock.  
Unmounting doesn’t work because the kernel already thinks it’s unmounted.  
The only fix is to **delete the directory itself**, which forces the kernel to drop the stale inode.

This is standard FUSE behavior.
