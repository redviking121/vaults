Xanadu Prime — Topology Map (Stub v0.1)
1. Purpose
The Topology Map defines the structural layout of the Prime substrate across:

repositories

directories

surfaces

governance layers

operational flows

This is the authoritative skeleton of the Prime architecture.
All future expansions must derive from this map.

2. High‑Level Org Topology
Prime consists of a single root organization with a governed set of repos.

Xanadu‑Prime — the root repo containing Prime‑era documents

Substrate‑Core — structural and ontological definitions

Ops‑Toolkit — scripts, scanners, and operator utilities

Demonstration‑Layer — public‑facing surfaces and render logic

Prime begins with Xanadu‑Prime only.
The others are declared but not yet instantiated.

3. Directory Topology (Prime Repo)
Inside Xanadu‑Prime, the directory structure is:

03_PRIME_ERA — all Prime‑era surfaces

substrate — incoming, normalized, and structural definitions

structure — indices, schemas, and views

ops — scripts, playbooks, operator utilities

tmp — ephemeral working space

backups — snapshot archives

These directories are governed surfaces.
They cannot be renamed or repurposed without a DSR‑1 event.

4. Surface Topology
Prime defines the following surface classes:

Constitutional Surfaces — axioms, governance, lifecycle

Structural Surfaces — substrate, indices, schemas

Public Surfaces — announcement, FAQ, explainer

Provenance Surfaces — authorship, custody, timeline

Operational Surfaces — ops scripts, workflows

Each surface class has:

a naming rule

a lifecycle rule

a propagation rule

This prevents drift and collapse.

5. Invariants
Prime topology is governed by the following invariants:

No cross‑region drift — surfaces cannot migrate between directories

No retroactive movement — once stabilized, a surface’s location is permanent

No implicit directories — all directories must be declared in the map

No ungoverned surfaces — every file must belong to a governed class

All surfaces must be declared and stabilized before propagation

These invariants are enforced by the substrate and DSR‑1.

6. Flow Topology
Prime defines three operational flows:

Creation Flow — declare → stabilize → propagate

Update Flow — modify → re‑stabilize → re‑propagate

Recovery Flow — restore → verify → reintegrate

Each flow is governed by Razor principles.
